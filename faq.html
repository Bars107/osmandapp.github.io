Why does OsmAnd require permission X

  • Camera/Voice Recording - this is only used by the Audio/Video notes plugin. This feature is optional and by default disabled (can be enabled in the Plugin Manager). It allows to quickly create location-related audio/video notes during a trip.
  • Read address book - is required by the app in order to use addresses that are defined in your address book.
  • GPS/Network access/SD card are natural requirements for this application.

How to build a route for a distance more than 250km?

Many long routes (> 200-250km) could not be built in OsmAnd today. If the app doesn't build it for 7-8 minutes, it is worth to consider puting waypoints in the middle of motorway. 3-4 waypoints will be enough to calculate even 1000km route.

Cannot use my SD Card with OsmAnd anymore? (4.4+)

If you updated your Android to version 4.4.2, then you are experiencing a known Android issue with WRITE_EXTERNAL_STORAGE permission. Android has changed regulation and now no applications could write to external card. Though if you have application running, it will continue working, but you won't be able to update any maps. Solution :
  • Move data to internal storage. Drawback: usually internal storage is very limited .
  • Move data to special folder on sd for OsmAnd+ - /storage/extSdCard/Android/data/net.osmand.plus and for OsmAnd /storage/extSdCard/Android/data/net.osmand/.
    Drawback: once you uninstall application all your data will be erased!

Route calculation is slow

Importantly, there are 2 route engines in the app: a Java based approach and a "Native" (C++) routing. The Java based approach is used in 'Safe Mode', it is 10x slower than native mode and it has strict memory limitations. If you experience it and you see messages 'Not enough memory to compute', please go to Settings → 'General' → 'Safe mode' and check the option is disabled.

For native routing there are different limitations for different phones, depending on memory & processor. In general, native routing should handle < 300 km route nicely. The route calculation should take from 15 sec till 4 minutes. It is not wise to wait longer 4 minutes, because most likely the program will crash.

The only known workaround to compute long routes is to put waypoints. 2 additional way points should be enough.

Routing in Osmand from start to destination point is wrong

In order to track down any routing issue that appears in Osmand, please open a new posting in Osmand googlegroup and tell us as detailed as possible these points:
  • What version of Osmand are you using, on what device?
  • Do you use online maps (tile graphics) or offline maps offered within Osmand app for download?
  • If you use offline maps, tell us the exact name of the map file where the routing issue occurs, and its edition date.
  • Tell us whether you have used Osmand's in-app offline routing, or any online routing provider like YOURS, OpenRouteService or OSRM.
  • What routing profile is choosen in Osmand app? (car, bike, foot)
  • How can we easily find start and end point within Osmand? If possible, tell us city name and street name for each. Also a Permalink from openstreetmap.org can be helpful.
  • Tell us your expected routing, and what Osmand is routing.
  • Try some other OSM based routing provider listed in the OSM wiki about [http://wiki.openstreetmap.org/wiki/Routing Routing] and investigate whether they fail also. Some online routing services offer a permalink, please post it also.

Osmand shows only some speed camera traps

Due to the geodata taken from the OpenStreetMap project there are by now two methods how speed camera traps are integrated in the raw OSM data:
  • One certain point (called "node" in OSM terminology) of a way is tagged with "highway=speed_camera") see OSM wiki about highway=speed_camera
  • a group of OSM data elements are joined together in a so called "relation" that contains more elements than a single node to describe the direction that is covered by the speed trap ... see Relation:enforcement
Currently, OsmAnd can only make use of the elements that consists of a single node. Analyzing of relations is to come in future releases.

What is the difference between tts (text to speech) and recorded voices?

TTS (text to speech) synthesizes a voice prompt from a specified text at the time it sounds, while recorded voices need to have every command snippet pre-recorded from a human voice.
While tts prompts may sound a bit more "robotic" and the language selection may be limited (both depends on the tts-engine installed on your device), many 'recorded' voices have various shortcomings (missing phrases, gaps) and cannot be easily maintained because you need access to the original speakers.

It is suggested to use a synthesized (tts) voice like en-tts. They are much more flexible, better maintained by the project developers. They are able to announce street names or highway numbers (version 1.5.1 and newer), etc., which recorded voices will likely never do.

The output quality of tts voices strictly depends on the tts engine you select (or install) in your Android device settings (built-in or 3rd party, try a few). I suggest that recorded voices should only be a fallback if you really like a particular voice, or if you cannot find a tts-engine supporting your language.

How to move OsmAnd home (maps) folder to external sd card?

In order to move OsmAnd home folder to an external sd card:
  • Go to Settings (on the start screen) → General → Storage directory.
  • Change the value to /storage/extSdCard
  • Move the OsmAnd folder to the external cd card using connection to the computer as an external drive or using file manager, like Astro or any other.

How to search address by a postcode

In order to find an address by a postcode:
  • Go to Search (on the start screen)
  • Move to the second tab on top (search by address)
  • Choose a Region
  • Click on the City button
  • Instead the city name fill in the postcode
  • Choose a Street
  • Choose a Building (not required)
Please keep in mind that OsmAnd data is based on the data from Open Street Map. This map is built by the Internet users. Sometimes in can be not complete. If you can't find the postcode you need, it does not always mean that the postcode is wrong. Maybe just nobody added it to the map. You can make this map better by adding the missing data. You can find more info about it here: OpenStreetMaps

House numbers are missing

OsmAnd uses Open Street Map (OSM) as a source of data.

This map is built by the Internet users. There are some advantages of such approach, like changes to the roads or POI appear very quickly, but there are also disadvantages, like missing data (house numbers as an example).

You can make this map better by adding the missing data. You can find more info about OSM and how to change it OpenStreetMaps and Guide

OsmAnd seems to drain too much battery power

There hav on and off been user reports about this. We made lots of measurements and could never really reproduce it, even though sometimes OsmAnd is reported of causing 90% of the battery usage. First of all OsmAnd doesn't run anything in background except Sleep mode but you need to enable it and you will always see the sleep mode icon in the status bar. So unless the sleep mode is active, if you exit the app (go to home screen or press close button), OsmAnd will stop working and will not cause any power consumption.

Power usage issues were reported with Android 2.3 and Android 4.0. In Android 4.0 you can double-check details of what causes how much battery use. If you check CPU time / screen time / GPS time consumed by OsmAnd (in android's standard power manager) you will see that the numbers don't add up correctly. For example, 2 min GPS, 2 min CPU, 1 min Screen causes 30% consumption. If you have some better good evidence, please contact to us.

PS: If you use OsmAnd for hiking or otherwise carry your device in a lose pocket where accidental pressing of the device's power switch may occur, it is also advisable to put OsmAnd to the background (by hitting the device's Home button), before putting the device to sleep. Reason: If OsmAnd's map screen is the foreground application, any accidental (unnoticed) push on the power button will not only activate the screen briefly (this is unavoidable), but will then also briefly activate GPS, which is quite a battery eater.

What do the 'Use current destination' and 'Pass along entire track' options mean when navigating using predefined GPX routes?

- "Use current destination": Guides you via the specified GPX-track, but to your currently selected "Destination", so the GPX route can be extended or shortened.

- "Pass along entire track": Forces the OsmAnd navigation algorithm to, as much as possible, traverse the entire GPX track, even if shortcuts are possible or you are already near its end when you start, or already rather close to your destination before traversing the entire GPX track.

Contour lines are not showing up?

Please follow these steps:
  • Check if the Contour Maps plugin you bought is enabled under Settings → Plugins → checkmark Contour Lines Plugin.
  • You need to download the contour maps from the menu in Settings → Data Management → Click Here To Download or Update Offline Data → tap the little triangle where it says "Regular Maps".
  • Make sure they are enabled in Menu → Map Layers → Hillshade Layer.
  • Check at what zoom level they appear in Menu → Configure Screen → Show contour lines.

Unfortunately, no. We need to generate completely new maps with different geometry and labels.

TTS doesn't function properly (i.e. Dutch).

If you are having issues with the TTS file, then please check if you have a compatible TTS engine installed on your Android device. It is a known issue that Android's default TTS engine (Pico) does not support Dutch. You can try to use something like SVOX or Espeak.

Offline search by address does not show all streets numbers

Most of these issues are map data issues and could be addressed via OpenStreetMaps. If you can see that data is present there (note that it takes some time for OSM new data to appear in OsmAnd maps), then problem is in OsmAnd algorithms or how that data is structured.

The most common issue is that streets exist but are not found in a city, and this is due to missing evidence in the OSM data to what town/city the street belongs. In OsmAnd search you can use feature "Search in neighboor cities" if you see that street is not found.

To investigate this problem you can use the BinaryInspector tool from OsmAndMapCreator installation which shows you exact information on what is in the obf file. Most probably you will find your street in a neighboring town. If you consider to open an issue, please specify all relevant information in its description.

If you want to see more deail about why these issues occur in OsmAnd's algorithms, please read Wiki

Creating own maps and OsmAndMapCreator

OsmAndMapCreator fails with message: OutOfMemoryError

The file you try to process with OsmAndMapCreator is big. Either, try to process a smaller file, or increase the memory for OsmAndMapCreator in the .sh or .bat file. The -Xmx parameter specifies how much memory the program can consume. Settings can be different for 64bit (more than 1.5GB) and 32bit (max around 1.5GB) machines.

I want to put some custom details on a vector map (create own rendering style with own data

OsmAndMapCreator can process only OSM files (osm-xml, bz2, pbf). However the set of tags can be custom. To specify what tags/values need to be indexed by Creator please download and change this file https://github.com/osmandapp/OsmAnd-core/blob/master/OsmAnd-java/src/net/osmand/osm/rendering_types.xml. OsmAndMapCreator has an option to use custom rendering_types.xml in the settings. Once file is created you can double check that data is present by utility binaryInspector with '-vmap' argument. This utility is packaged with OsmAndMapCreator.

Once the .obf file is ready you can create custom rendering file to display missing attributes. There is a default rendering style which contains all information about rendering https://github.com/osmandapp/OsmAnd-resources/blob/master/rendering_styles/default.render.xml. It is good to have a look at it but it is very hard to open/edit it and understand. More convenient way to create your own rendering style is to create style that depends (inherits) default style. A good example of custom rendering style is a https://github.com/osmandapp/OsmAnd-resources/blob/master/rendering_styles/Winter-and-ski.render.xml. See also [http://code.google.com/p/osmand/wiki/CustomizeOBFFileContent How to produce customized vector data].

Currently OsmAndMapCreator doesn't support relation tagging. So you need to manually copy all tags from relations (like route color) to way tags by script.

Beta Testing

The Google Play Store has a possibility to test beta versions of apps, which enables you to (temporary) download and use the test version from the Google Play Store instead of the standard available version(s). For OsmAnd this means that you can participate in testing the new OsmAnd free or the new OsmAnd+ paid versions. This enables you to use and test exciting new features. Using a beta version also means that you might encounter bugs or app crashes. The Google Play Store mechanism also supports the easy return to the standard version from the Google Play Store.

Activating the test version

Register as tester for the OsmAnd free test version via the following Google Group: https://play.google.com/apps/testing/net.osmand Register for the Osmand+ paid test version via the following Google Group: Testing (note: this is only possible if you are a registered user for the paid version)

  • When you click the link and you register yourself, you see a big blue or green panel mentioning "Be a test user" (in your defined language).
  • After having clicked the "button" you will see a message "You are now a tester" with a green check-mark in front of it (and on repetitive actions it will show "You are a tester").
  • Give Google some time to synchronize which can be anywhere between 10 seconds or 60 minutes.
  • Now you can download the test version, either directly from the Google Play Store, or via your phone/tablet.

Returning to the standard Google Play Store OsmAnd version

Note that the same registration screen, after having registered, also has a link "Leave the test version". If, after some time, you want to use the current standard version from the Google Play Store again instead of the test version, you do the following:
  • Use the relevant link above again (so you better make this a bookmark/favorite in your browser). That page will still mention "You are a tester"
  • That page also contains the link "Leave the test version".
  • Click that link and Google will mention "You have left the test version"
  • Manually uninstall the test version, either via the Google Play Store or on your phone/tablet.
  • Manually (re)install the current version, either via the Google Play Store or on your phone/tablet.
Note: Make an export of your OsmAnd favorites. The new testversion should simply continue using the favorites but as it is a beta version for testing, you never know.

OsmAnd Development

How can I contribute a new language TTS voice file?

The TTS voice files stored here Github voice. You can fork a repository and then create pull request with your file.
  • Please do not use an underscore _ at the beginning of your file name.
  • Do not change version in the ttsconfig.p, use the one that is in the others. This does not denote a file version, but rather an API version that the tts uses.


Please refer to our English en-tts file for basic reference and as template, and consider to look at grammatically more complex languages like German (de-tts), Czech (cz-tts) or Slovak (sk-tts) to cover more complex statements. There should be a way to correctly produce what needs to be done - the PROLOG used in the tts-config.p files is a powerful programming language.

Regarding distribution: In the latest versions the tts voices are distributed within the app (apk) and need not / cannot be downloaded any more via the download server on the web. (Don't be confused, they still appear like downloads in the app's Offline Data Manager, but they are listed there only and contained locally). However, until the next release of OsmAnd we could put new tts-voices on our googlecode Download for manual download. The basic idea is that tts files will not be updated too frequently, so in general it is sufficient to have them distributed with each release, within the application.
