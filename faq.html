Some online routing services offer a permalink, please post it also. == Route calculation is very slow == First of all there are 2 route engines in the program: a Java based approach and a "Native" (C++) routing. The Java based approach is used in 'Safe Mode' and in 'Precise routing (alpha)' and is 10x slower than native mode and it has strict memory limitations. If you experience it and you see messages 'Not enough memory to compute', please go to Settings -> 'General' -> 'Safe mode' and validate it is disabled. For native routing there are different limitations for different phones, depending on memory & processor. In general, native routing should handle < 300 km route nicely. The route calculation should take from 15 sec till 4 minutes. It is not wise to wait longer 4 minutes, because most likely the program will crash. The only known workaround to compute long routes is to put waypoints. 2 additional way points should be enough. == Osmand does not show all speed camera traps, only some of them == Due to the geodata taken from the OpenStreetMap project there are by now two methods how speed camera traps are integrated in the raw OSM data: * One certain point (called "node" in OSM terminology) of a way is tagged with "highway=speed_camera" ... see OSM wiki about [http://wiki.openstreetmap.org/wiki/Tag%3Ahighway%3Dspeed_camera highway=speed_camera] or * a group of OSM data elements are joined together in a so called "relation" that contains more elements than a single node to describe the direction that is covered by the speed trap ... see [http://wiki.openstreetmap.org/wiki/Relation:enforcement Relation:enforcement] Currently, `OsmAnd` can only make use of the elements that consists of a single node. Analyzing of relations is to come in future releases. == Voice navigation does not work with the display is off == Most of the time, the problem is that people don't know they have to turn on `OsmAnd's 'Sleep mode functionality' to allow the phone to process events in the background. Without this, `OsmAnd loses the GPS fix and cannot give you accurate instructions. For the best results, set the Wake-up interval to 0 seconds (continuous mode). If this still does not work for you, please search googlecode and our forum for similar bugs, then ask on our forum. ==What is the difference between tts (text to speech) and recorded voices?== TTS (text to speech) synthesizes a voice prompt from a specified text at the time it sounds, while recorded voices need to have every command snippet pre-recorded from a human voice. While tts prompts may sound a bit more "robotic" and the language selection may be limited (both depends on the tts-engine installed on your device), many 'recorded' voices have various shortcomings (missing phrases, gaps) and cannot be easily maintained because you need access to the original speakers. I suggest you use a synthesized (tts) voice like en-tts. They are much more flexible, better maintained by the project developers. They are able to announce street names or highway numbers (version 1.5.1 and newer), etc., which recorded voices will likely never do. The output quality of tts voices strictly depends on the tts engine you select (or install) in your Android device settings (built-in or 3rd party, try a few). I suggest that recorded voices should only be a fallback if you REALLY like a particular voice, or if you cannot find a tts-engine supporting your language. == Why does `OsmAnd` not offer access to google maps? == Basically, there are two reasons. We made lots of measurements and could never really reproduce it, even though sometimes `OsmAnd` is reported of causing 90% of the battery usage. First of all `OsmAnd` doesn't run anything in background except Sleep mode but you need to enable it and you will always see the sleep mode icon in the status bar. So unless the sleep mode is active, if you exit the app (go to home screen or press close button), `OsmAnd` will stop working and will not cause any power consumption. Power usage issues were reported with Android 2.3 and Android 4.0. In Android 4.0 you can double-check details of what causes how much battery use. If you check CPU time / screen time / GPS time consumed by `OsmAnd` (in android's standard power manager) you will see that the numbers don't add up correctly. For example, 2 min GPS, 2 min CPU, 1 min Screen causes 30% consumption. If you have some better good evidence, please contact to us. PS: If you use `OsmAnd` for hiking or otherwise carry your device in a lose pocket where accidental pressing of the device's power switch may occur, it is also advisable to put `OsmAnd` to the background (by hitting the device's Home button), before putting the device to sleep. Reason: If `OsmAnd's` map screen is the foreground application, any accidental (unnoticed) push on the power button will not only activate the screen briefly (this is unavoidable), but will then _also_ briefly activate GPS, which is quite a battery eater. == Why does `OsmAnd` require permission X == * Camera/Voice Recording - this is only used by the Audio/Video notes plugin. It allows to quickly create location-related audio/video notes during a trip. * Read address book - is required by the app in order to use addresses that are defined in your address book * GPS/Network access/SD card are natural requirements for this application == What do the 'Use current destination' and 'Pass along entire track' options mean when navigating using predefined GPX routes? == - "Use current destination": Guides you via the specified GPX-track, but to your currently selected "Destination", so the GPX route can be extended or shortened. - "Pass along entire track": Forces the `OsmAnd` navigation algorithm to, as much as possible, traverse the entire GPX track, even if shortcuts are possible or you are already near its end when you start, or already rather close to your destination before traversing the entire GPX track. == I've got NaN in my google docs using the Live tracking feature == Make sure the url in live tracking has the correct format, having {0} as parameters and not encoded {,} with percentages. Splitting is necessary due to the max file size limit on googlecode. To extract the file: # FIRST concatenate the files together, # THEN unzip them. TotalCommander can combine files under Windows. Under Linux and Mac OS X you can use cat >> notation, Hjsplit, or any other merge utility. If you use `cat` to combine the zip files, you need to fix the merged zip file first. The fixed merged file can be unzipped as a usual zip file.
Example (Gb_england consists of three partial zip files): {{{ cat Gb_england_europe_2.obf.zip-* > Gb_england_europe_2.obf.zip zip -FF Gb_england_europe_2.obf.zip --out Gb_england_europe_2.obf-full.zip unzip -o Gb_england_europe_2.obf-full.zip }}} == Offline search by address does not show all streets/house numbers == Most of these issues are map data issues and could be addressed via http://www.openstreetmap.org/. If you can see that data is present there (note that it takes some time for OSM new data to appear in `OsmAnd` maps), then problem is in `OsmAnd` algorithms or how that data is structured. The most common issue is that streets exist but are not found in a city, and this is due to missing evidence in the OSM data to what town/city the street belongs. To investigate this problem you can use the `BinaryInspector` tool which shows you exact information on what is in the obf file, see [http://code.google.com/p/osmand/wiki/HowToInspectAnObfBinaryFile]. Most probably you will find your street in a neighboring town. If you consider to open an issue, please open it in the right place and specify all relevant information in its description. If you want to see more deail about why these issues occur in `OsmAnd's` algorithms, please read [http://code.google.com/p/osmand/wiki/MapAddressDataStructure]. == I installed `OsmAnd` Version 0.8.x, and some vector maps report they are not supported by this version of `OsmAnd` == How did you install the map? You can fork a repository and then create pull request with your file. * Please do not use an underscore _ at the beginning of your file name. * Do not change version in the ttsconfig.p, use the one that is in the others. This does not denote a file version, but rather an API version that the tts uses. Please refer to our English en-tts file for basic reference and as template, and consider to look at grammatically more complex languages like German (de-tts), Czech (cz-tts) or Slovak (sk-tts) to cover more complex statements. There should be a way to correctly produce what needs to be done - the PROLOG used in the tts-config.p files is a powerful programming language. Regarding distribution: In the latest versions the tts voices are distributed within the app (apk) and need not / cannot be downloaded any more via the download server on the web. (Don't be confused, they still appear like downloads in the app's Offline Data Manager, but they are listed there only and contained locally). However, until the next release of `OsmAnd` we could put new tts-voices on our googlecode Download for manual download. The basic idea is that tts files will not be updated too frequently, so in general it is sufficient to have them distributed with each release, within the application. = Beta Testing = The Google Play Store has a possibility to test beta versions of apps, which enables you to (temporary) download and use the test version from the Google Play Store instead of the standard available version(s). For `OsmAnd` this means that you can participate in testing the new `OsmAnd` free or the new `OsmAnd+` paid versions. This enables you to use and test exciting new features. Using a beta version also means that you might encounter bugs or app crashes. The Google Play Store mechanism also supports the easy return to the standard version from the Google Play Store. == Activating the test version == Register as tester for the `OsmAnd` free test version via the following Google Group: https://play.google.com/apps/testing/net.osmand Register for the `Osmand+` paid test version via the following Google Group: https://play.google.com/apps/testing/net.osmand.plus/testing
(note: this is only possible if you are a registered user for the paid version) * When you click the link and you register yourself, you see a big blue or green panel mentioning "Be a test user" (in your defined language). * After having clicked the "button" you will see a message "You are now a tester" with a green check-mark in front of it (and on repetitive actions it will show "You are a tester"). * Give Google some time to synchronize which can be anywhere between 10 seconds or 60 minutes. * Now you can download the test version, either directly from the Google Play Store, or via your phone/tablet. == Returning to the standard Google Play Store `OsmAnd` version == Note that the same registration screen, after having registered, also has a link "Leave the test version". If, after some time, you want to use the current standard version from the Google Play Store again instead of the test version, you do the following: * Use the relevant link above again (so you better make this a bookmark/favorite in your browser). That page will still mention "You are a tester". * That page also contains the link "Leave the test version". * Click that link and Google will mention "You have left the test version" * Manually uninstall the test version, either via the Google Play Store or on your phone/tablet. * Manually (re)install the current version, either via the Google Play Store or on your phone/tablet. _Note:_ Make an export of your `OsmAnd` favorites. The new testversion should simply continue using the favorites but as it is a beta version for testing, you never know.
